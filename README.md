# 智能桌面系统

基于MCP（Model Context Protocol）协议的智能桌面助手，支持文档转换、文件操作、邮件处理、网络请求等多种桌面任务的自动化处理。

## ✨ 特性

- 🧠 **智能意图识别**：自动理解用户需求，支持三种意图类型（task/chat/cannot_execute）
- 📋 **任务规划**：基于行为树的任务规划系统，支持复杂任务的自动分解和执行
- 🚀 **高效缓存**：基于FAISS向量搜索的混合缓存系统，支持哈希精确匹配和语义匹配
- 🔒 **安全可靠**：沙箱隔离、权限控制
- 🧩 **易于扩展**：模块化设计，支持自定义工具
- ☁️ **天气查询**：实时天气信息查询和预警通知
- 📄 **批量文档处理**：支持批量文件转换和通配符匹配
- 💭 **LLM思考过程**：可折叠的思考过程显示

## 🚀 快速开始

### 环境要求

- Python 3.13+
- Ollama服务
- 必要的Python库（见requirements.txt）

### 安装步骤

```bash
# 1. 克隆仓库
git clone https://github.com/XuChang-robot/Intelligent-Desktop.git
cd Intelligent-Desktop

# 2. 安装依赖
pip install -r requirements.txt

# 3. 启动Ollama服务
ollama serve

# 4. 下载模型
ollama pull qwen3:8b
ollama pull nomic-embed-text

# 5. 启动系统
python main.py
```

### 行为树任务规划

行为树是一种用于任务规划和执行的高级架构，通过树状结构来表示任务的执行逻辑。

#### 行为树的优势

- **模块化设计**：每个节点都是独立的模块，可以单独开发和测试
- **灵活的执行流程**：支持条件分支、循环、并行执行等复杂控制流
- **易于扩展**：可以轻松添加新的节点类型和功能
- **可视化和调试**：可以直观地查看任务执行流程，便于调试和优化
- **错误处理**：任务执行失败时可以自动回退或重试
- **状态管理**：实时跟踪任务执行状态和进度

#### 行为树运行方式举例

假设用户请求："帮我整理桌面上的PDF文件，将它们转换成Word格式，然后发送到指定邮箱"

行为树会将这个任务分解为以下步骤：

1. **根节点**：开始执行任务
2. **序列节点**：按顺序执行以下子任务
   - **搜索节点**：在桌面目录搜索所有PDF文件
   - **循环节点**：对每个找到的PDF文件执行以下操作
     - **转换节点**：将PDF文件转换为Word格式
   - **收集节点**：收集所有转换后的Word文件
   - **发送节点**：将收集的Word文件发送到指定邮箱

每个节点都有自己的执行逻辑和错误处理机制，如果某个节点执行失败，行为树会根据预设的策略进行处理（如重试、跳过或终止任务）。

#### 当前缓存逻辑

系统采用两层缓存匹配策略来提高任务规划的效率：

1. **第一层：哈希精确匹配**
   - 基于请求内容的MD5哈希进行精确匹配
   - 完全相同的请求会立即命中缓存
   - 响应速度最快（约0.01秒）
   - 适用于：用户重复执行相同的任务

2. **第二层：FAISS语义匹配**
   - 基于向量相似度进行语义匹配
   - 相似但表述不同的请求可以命中缓存
   - 响应速度较快（约0.05秒）
   - 受`similarity_threshold`参数控制（默认0.85）
   - 适用于：用户用不同的表述方式执行相似的任务

#### 缓存的好处

- **提高响应速度**：避免重复调用LLM生成计划，显著减少响应时间
- **降低资源消耗**：减少LLM API调用次数，节省计算资源和费用
- **保证一致性**：相同或相似的任务使用相同的执行计划，提高系统稳定性
- **支持离线模式**：缓存命中时可以在不连接LLM服务的情况下执行任务

## ⚙️ 配置

### 用户配置

用户配置文件位于 `user_config/config.yaml`，包含以下配置项：

- **MCP配置**：服务器地址、端口、连接数
- **LLM配置**：模型名称、API地址、温度参数
- **安全配置**：沙箱、危险命令、权限控制
- **UI配置**：窗口标题、大小、主题
- **日志配置**：日志级别、格式
- **邮件配置**：SMTP/IMAP服务器
- **缓存配置**：启用状态、大小限制、相似度阈值

详细配置说明请参考：[用户配置说明](user_config/README.md)

### 系统配置

系统配置文件位于 `system_config/cache_config.json`，包含缓存关键参数配置。

**注意**：系统配置由系统维护，用户不应该修改。

详细配置说明请参考：[系统配置说明](system_config/README.md)
