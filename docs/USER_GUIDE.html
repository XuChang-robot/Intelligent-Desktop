系统概述

智能桌面系统是一个基于MCP（Model Context Protocol）架构的智能助手，可以帮助用户完成各种桌面任务，包括文档转换、文件操作、邮件处理、网络请求等。

核心特性

  • 智能意图识别：自动理解用户需求
  • 任务规划：自动生成执行计划
  • 高效缓存：基于FAISS向量搜索的混合缓存系统
  • 安全可靠：沙箱隔离、权限控制
  • 易于扩展：模块化设计，支持自定义工具

缓存检查

    意图类型

    系统支持两种意图类型：

    1. task（任务型）

    需要调用工具执行具体操作，包括：

    • 文件操作：创建、读取、写入、删除文件/文件夹
    • 文档转换：PDF转Word、Word转PDF等
    • 系统信息：获取系统信息、进程列表等
    • 文本处理：转语音、摘要、格式化等
    • 网络请求：HTTP请求、API调用等
    • 邮件发送：发送邮件、读取邮件等

    2. chat（聊天型）

    纯对话内容，直接使用LLM回答，包括：

    • 问候：如"你好"、"早上好"等
    • 提问：如"什么是"、"如何"、"为什么"等
    • 闲聊：如"今天天气怎么样"、"你好吗"等
    • 咨询：各种问题咨询和建议

    快速开始

    安装依赖

    pip install -r requirements.txt

    启动系统

    python main.py

    配置指南

    配置文件结构

    Intelligence_Desktop/
├── system_config/           # 系统配置（用户不应该修改）
│   ├── cache_config.json    # 缓存关键参数配置
│   └── README.md          # 系统配置说明
├── user_config/            # 用户配置
│   ├── config.yaml         # 用户配置文件
│   ├── config.py          # 配置加载器
│   ├── README.md          # 用户配置说明
│   └── CACHE_CONFIG_GUIDE.md  # 缓存配置指南

    MCP配置

    mcp:
  server:
    host: "localhost"           # MCP服务器地址
    port: 8001                   # MCP服务器端口
    max_connections: 10          # 最大连接数
  client:
    retry_attempts: 3            # 客户端重试次数

    LLM配置

    llm:
  model: "qwen3:8b"             # LLM模型名称
  base_url: "http://localhost:11434"  # LLM服务地址
  temperature: 0.6              # 温度参数（0-1，越高越随机）
  max_tokens: 4000              # 最大生成token数

    安全配置

    security:
  enable_sandbox: true          # 是否启用沙箱
  dangerous_commands: ["rm -rf", "format", "shutdown", "reboot"]  # 危险命令列表
  allow_network: true           # 是否允许网络操作
  allow_file_system: true       # 是否允许文件系统操作

    UI配置

    ui:
  title: "智能桌面系统"         # 窗口标题
  width: 800                    # 窗口宽度
  height: 600                   # 窗口高度
  theme: "light"                # 主题（light/dark）

    缓存配置

    cache:
  enabled: true                  # 是否启用缓存
  type: "faiss"                 # 缓存类型
  cache_dir: "cache"            # 缓存目录

  ttl: 604800                   # 缓存过期时间（秒），默认一周
  max_total_size_mb: 1024       # 总大小限制（1GB）
  max_db_size_mb: 768           # SQLite数据库最大768MB
  max_faiss_size_mb: 392       # FAISS索引最大392MB
  max_records: 100000           # 最多100000条记录

  cleanup_interval: 3600        # 清理间隔（秒），默认1小时
  cleanup_on_startup: true     # 启动时清理过期缓存

  similarity_threshold: 0.85    # 相似度阈值（0-1）
  embedding_model: "nomic-embed-text"  # Embedding模型

    #### 缓存配置说明

    TTL（过期时间）：

  • 604800秒（7天）：默认设置，适合大多数场景
  • 3600秒（1小时）：适合频繁更新的内容
  • 86400秒（24小时）：适合稳定的内容
    • 大小限制：

  • max_total_size_mb：总大小限制，超过后会清理旧缓存
  • max_db_size_mb：SQLite数据库大小限制，建议设置为总大小的75%
  • max_faiss_size_mb：FAISS索引大小限制，建议设置为总大小的25%
    • 相似度阈值：

  • 0.85：平衡匹配（推荐）
  • 0.90：高精度匹配，缓存命中率较低
  • 0.80：宽松匹配，缓存命中率较高

    • 详细配置指南请参考：缓存配置指南

    常见问题

    Q1: 意图解析失败怎么办？

    A: 检查以下几点：

    1. 确保LLM服务正常运行

    2. 检查LLM配置是否正确

    3. 查看日志了解具体错误信息

    Q2: 缓存未命中怎么办？

    A: 检查以下几点：

    1. 确保缓存已启用

    2. 检查相似度阈值是否过高

    3. 查看日志了解缓存状态

    Q3: 如何提高缓存命中率？

    A: 可以尝试以下方法：

    1. 降低相似度阈值（如从0.85降到0.80）

    2. 增加缓存过期时间

    3. 使用更相似的表述

    Q4: 如何清理缓存？

    A: 有以下几种方法：

    1. 自动清理：系统会定期清理过期缓存

    2. 手动清理：删除cache目录

    3. 调整配置：减少TTL时间

    Q5: 系统支持哪些文件格式？

    A: 支持以下文件格式：

  • 文档：PDF、Word（.docx）
  • 文本：TXT
  • 音频：MP3、WAV（文字转语音）
  • 图片：PNG、JPG（OCR识别）
