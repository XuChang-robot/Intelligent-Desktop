混合缓存系统使用说明

概述

混合缓存系统结合了GPTCache的语义匹配能力和文件持久化存储的优势，为智能桌面系统提供高效的任务计划缓存功能。

为什么使用缓存系统

1. 性能优化

LLM生成任务计划是一个耗时的过程，通常需要几秒钟的时间。通过缓存系统，可以显著减少响应时间：

性能对比（Intel® Core™ i9-13980HX + NVIDIA® GeForce RTX™ 4090 Laptop）：

  • 精确匹配：缓存命中时响应时间从7.7秒降至0.01秒，性能提升770倍
  • 语义匹配：相似请求的响应时间从7.7秒降至0.05秒，性能提升154倍
  • 用户体验：大幅提升系统响应速度，提供更流畅的用户体验

注意：实际性能提升可能因硬件配置、LLM模型、网络环境等因素而有所不同。上述数据基于特定测试环境，仅供参考。

2. 成本节约

LLM API调用通常需要付费，缓存系统可以有效降低使用成本：

  • 减少API调用：缓存命中时无需调用LLM，大幅降低API使用量
  • 节省费用：对于重复或相似的请求，可以节省大量API调用费用
  • 资源优化：减少对LLM服务的依赖，优化系统资源使用

3. 系统可靠性

缓存系统提高了系统的可靠性和稳定性：

  • 离线能力：即使LLM服务不可用，缓存中的请求仍然可以正常响应
  • 降级处理：当LLM服务出现问题时，系统可以依赖缓存继续运行
  • 减少依赖：降低对外部LLM服务的依赖，提高系统自主性

4. 智能匹配

传统的缓存系统只能进行精确匹配，而混合缓存系统支持语义匹配：

  • 理解语义：能够识别语义相似的请求，即使表述不同也能命中缓存
  • 提高命中率：相比精确匹配，语义匹配可以大幅提高缓存命中率
  • 灵活性：用户可以用不同的方式表达相同的意图，系统仍能识别

5. 持久化存储

混合缓存系统支持持久化存储，提供更好的数据管理：

  • 重启有效：系统重启后缓存仍然有效，无需重新生成
  • 长期存储：缓存可以长期保存，适用于重复性高的任务
  • 自动管理：支持TTL自动清理，避免缓存无限增长

6. 实际应用场景

缓存系统在以下场景中特别有价值：

  • 重复任务：用户经常执行相同的任务（如发送固定格式的邮件）
  • 相似任务：用户执行相似的任务（如发送不同内容的邮件）
  • 批量操作：批量处理相似的任务时，缓存可以大幅提升效率
  • 测试调试：开发和测试过程中，避免重复调用LLM

架构设计

用户请求
    ↓
GPTCache (语义匹配)
    ↓ 缓存命中
返回结果
    ↓ 缓存未命中
调用LLM
    ↓
GPTCache + 文件持久化 (双重存储)
    ↓
返回结果

核心特性

1. 三层缓存机制

  • 第一层：GPTCache语义匹配（内存缓存）
  • 第二层：文件精确匹配（磁盘缓存）
  • 第三层：文件语义匹配（磁盘缓存）

2. 智能匹配策略

  • 精确匹配：基于MD5哈希的精确匹配
  • 语义匹配：基于向量相似度的语义匹配
  • 输入归一化：自动处理标点、空格等细微差异

3. 持久化存储

  • 文件存储：所有缓存都保存到磁盘
  • 重启有效：系统重启后缓存仍然有效
  • 自动过期：支持TTL（Time To Live）自动清理

配置说明

在config/config.yaml中配置缓存系统：

# 缓存配置
cache:
  enabled: true
  type: "hybrid"  # 混合缓存：GPTCache + 文件持久化
  cache_dir: "cache"
  ttl: 3600  # 缓存过期时间（秒）
  similarity_threshold: 0.85  # 语义相似度阈值
  # GPTCache配置
  gptcache:
    enabled: true
    embedding_model: "onnx"  # 使用ONNX模型生成向量
    data_manager: "map"  # 内存管理器
  # 文件缓存配置
  file_cache:
    enabled: true
    auto_cleanup: true  # 自动清理过期缓存
    cleanup_interval: 3600  # 清理间隔（秒）

使用方法

1. 初始化TaskPlanner

from mcp_client.task_planner import TaskPlanner
from mcp_client.llm import LLMClient
from config.config import load_config

# 加载配置
config = load_config()

# 初始化LLM客户端
llm_client = LLMClient()

# 初始化任务规划器（带混合缓存）
cache_config = config.get('cache', {})
task_planner = TaskPlanner(
    llm_client=llm_client,
    cache_dir=cache_config.get('cache_dir', 'cache'),
    cache_ttl=cache_config.get('ttl', 3600),
    similarity_threshold=cache_config.get('similarity_threshold', 0.85)
)

2. 使用缓存的任务规划

# 规划任务（自动使用缓存）
intent = {
    "intent": "task",
    "entities": {
        "收件人邮箱": "457911161@qq.com",
        "邮件主题": "演示",
        "邮件内容": "hello，项目地址是：https://github.com/XuChang-robot/Intelligent-Desktop.git"
    },
    "confidence": 0.9
}
plan = await task_planner.plan_task(intent)

3. 缓存管理

# 获取缓存统计信息
stats = task_planner.get_cache_stats()
print(f"总文件数: {stats['total_files']}")
print(f"有效文件数: {stats['valid_files']}")
print(f"过期文件数: {stats['expired_files']}")
print(f"总大小: {stats['total_size_mb']} MB")

# 清理过期缓存
cleaned = task_planner.cleanup_expired_cache()
print(f"清理了 {cleaned} 个过期缓存文件")

# 清空所有缓存
task_planner.clear_cache()

高级配置

调整相似度阈值

cache:
  similarity_threshold: 0.90  # 更严格的匹配
  • 0.80-0.85：宽松匹配，更多缓存命中
  • 0.85-0.90：平衡匹配（推荐）
  • 0.90-0.95：严格匹配，更少误匹配

调整缓存过期时间

cache:
  ttl: 7200  # 2小时
  • 1800：30分钟（适合频繁更新的内容）
  • 3600：1小时（推荐）
  • 7200：2小时（适合稳定的内容）
  • 86400：24小时（适合很少更新的内容）

禁用GPTCache

如果GPTCache初始化失败，系统会自动降级到文件缓存：

cache:
  gptcache:
    enabled: false  # 禁用GPTCache

故障排除

GPTCache初始化失败

问题：日志显示"GPTCache初始化失败"

解决方案：

  • 检查是否安装了gptcache：pip install gptcache
  • 检查ONNX模型是否可用
  • 系统会自动降级到文件缓存

嵌入模型加载失败

问题：日志显示"嵌入模型加载失败"

解决方案：

  • 检查是否安装了sentence-transformers：pip install sentence-transformers
  • 系统会使用精确匹配和文件缓存

缓存文件过大

问题：缓存目录占用过多磁盘空间

解决方案：

  • 定期清理过期缓存：task_planner.cleanup_expired_cache()
  • 减少TTL时间
  • 清空所有缓存：task_planner.clear_cache()

最佳实践

  • 定期清理：设置自动清理过期缓存
  • 监控统计：定期查看缓存统计信息
  • 调整参数：根据实际使用情况调整相似度阈值和TTL
  • 测试验证：使用测试脚本验证缓存效果
